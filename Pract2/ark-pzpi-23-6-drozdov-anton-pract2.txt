3 ОПИС ВИКОНАНОЇ РОБОТИ
        3.1 Рефакторинг
        Рефакторинг коду - це процес покращення структури коду без якихось функціональних змін. Рефакторинг робиться з метою покращення структури коду для покращення його розуміння у команді, для зниження порогу входу для нових членів команди, для того щоб зробити код простішим у підтримці та розширенні, щоб зробити подальшу розробку більш ефективною.
        Під час виконання роботи було розглянуто наступні методи рефакторингу: Remove Middle Man, Replace Error Code with Exception та Pull Up Method. Для прикладів коду у роботі використовується мова Java.
        3.2 Дослідження методів рефакторингу
        Remove Middle Man
        Розглянемо випадок з класом-посередником, методи якого просто викликають методи вкладеного класу, не надаючи ніякої додаткової логіки.
        Згідно з методом Remove Middle Man такий клас або повністю видаляється (якщо в нього крім цих делегованих викликів більше нічого немає), або повертає об’єкт-делегат і дає можливість клієнту класу викликати необхідні функції самостійно.
        Цей метод є протилежний метод Hide Delegate і його треба використовувати обережно, не порушуючи закон Деметри. Якщо клієнту потрібно лише одне/два значення - то краще цей метод не застосовувати.
До рефакторингу:
1. class Person {
2.    Department department;
3. 4.    public Person getManager() {
5.        return department.getManager();
6.    }
7. 8.    public Person getHead() {
9.        return department.getHead();
10.    }
11. 12.    public String getDepartmentName() {
13.        return department.getName();
14.    }
15. }
16. 17. class Department {
18.    private Person manager;
19.    private Person head;
20.    private String name;
21. 22.    public Person getManager() {
23.        return manager;
24.    }
25. 26.    public Person getHead() {
27.        return head;
28.    }
29. 30.    public String getName() {
31.        return name;
32.    }
33. }
34. 35. Person manager = john.getManager();
36. String deptName = john.getDepartmentName();
37. Person head = john.getHead();

Replace Error Code with Exception
Замість повернення числових кодів в якості результату функції при помилках, функції повинні генерувати винятки (exception). Це робиться для відокремлення бізнес-логіки від обробки помилок та для використання єдиного уніфікованого методу обробки помилок у системі.
До рефакторингу:
1. class Account {
2.    private double balance;
3.    private boolean isBlocked;
4. 5.    public int withdraw(double amount) {
6.        if (isBlocked) {
7.            return -1;
8.        }
9.        if (amount > balance) {
10.            return -2;
11.        }
12.        balance -= amount;
13.        return 0;
14.    }
15. }
16. 17. int result = account.withdraw(100);
18. if (result == -1) {
19.    // Обліковий запис заблоковано
20. } else if (result == -2) {
21.    // Недостатньо коштів
22. } else {
23.    // Виведення успішне
24. }
Після рефакторингу:
1. class AccountException extends RuntimeException {}
2. class InsufficientFundsException extends AccountException {}
3. class AccountBlockedException extends AccountException {}
4. 5. class Account {
6.    private double balance;
7.    private boolean isBlocked;
8. 9.    public void withdraw(double amount) {
10.        if (isBlocked) {
11.            throw new AccountBlockedException();
12.        }
13.        if (amount > balance) {
14.            throw new InsufficientFundsException();
15.        }
16.        balance -= amount;
17.        return 0;
18.    }
19. }
20. 21. try {
22.    account.withdraw(500);
23.    // Виведення успішне
24. } catch (InsufficientFundsException e) {
25.    // Недостатньо коштів
26. } catch (AccountBlockedException e) {
27.    /* Обліковий запис заблоковано */ }
Pull Up Method
        Цей метод застосовується, коли декілька підкласів мають методи, що виконують однакову роботу. Такі методи треба винести у батьківський клас, таким чином прибравши дублювання коду. Якщо методи не ідентичні, а просто схожі, то спочатку треба однакові частини методів винести в окремий метод (Extract Method метод), а вже потім підняти такий метод у батьківський клас.
        До рефакторингу:
1. class Employee {}
2. 3. class Salesman extends Employee {
4.    private String name;
5.    public String getName() { return name; }
6. }
7. 8. class Engineer extends Employee {
9.    private String name;
10.    public String getName() { return name; }
11. }
        Після рефакторингу:
1. class Employee {
2.    protected String name;
3.    public String getName() { return name; }
4. }
5. 6. class Salesman extends Employee {}
7. class Engineer extends Employee {}
